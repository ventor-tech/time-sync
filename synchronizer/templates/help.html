Quick start guide

This guide will help you to synchronize your worklogs first time. We will synchronize worklogs from Toggl to JIRA step-by-step.

0. Prepare to synchronization

In order to synchronize your worklogs from Toggl you need fill descriptions of worklogs in specific format: 

[jira:TIMESYNC-8] Do not require connector name in case one target source
[TIMESYNC-8] Do not require connector name in case one target source *
* Valid when Default Target is specified. For details check chapter 3.

Toggl worklogs that have a different format will not be synchronized! We will synchronize the following Toggl worklogs:

1. Create source and target connectors

To synchronize worklogs we need at least two connectors:

  • Source connector
  • Target connector

Click to Connectors button in main menu: Click on Add a new one button

Fill all required fields in the following form: Examples

  1. For Toggl you need to fill fields:
    • Connector Name
    • API Token
    • Select connector type
  2. For JIRA you need to fill fields:
    • Connector Name
    • Server (Without http/https! Example: jira.test.org)
    • Login
    • API Token
    • Select connector type

2. Configure your account

To configure your account click on your username in right top corner of menu. Set correct values for the following fields: NOTE. Default target will simplify format of worklogs description in Toggl to: 

[TIMESYNC-8] Do not require connector name in case one target source
In this case you can omit connector name - app will set it to Default target value automatically.

3. Your first synchronization

Click on logo or Worklogs menu item. Then click on Synchronize now button. You will see the following form: Set your new Toggl connector as Source and JIRA as Target. Select date from which you want to synchronize worklogs. Click Save
In next page you will see table with green (valid) and red (invalid, these will not be synchronized!) worklogs. In this step you can Edit or Delete worklogs with corresponding buttons.
Then click Save worklogs button to run synchronization. In next page you will see synchronization results: Let's check that worklogs were synchronized:
